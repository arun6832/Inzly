# Inzly: The Innovation Execution Hub

**Inzly** is a high-impact platform designed to bridge the gap between initial idea discovery and real-world execution. Unlike traditional ideation tools that stop at discovery, Inzly provides a tactical workspace where ideas are validated, refined, and moved through a structured evolution path toward building and launching.

---

## 🚀 The Core Philosophy: "Idea → Execution"

Inzly operates on a **Problem-First** architecture. We believe that lasting innovation starts with a deep understanding of local or global challenges.

### 🏛️ Refinement Hub & Evolution Timeline
Concept maturity is tracked through a vertical evolution timeline. 
- **Version Snapshots**: Every iteration of an idea is snapshotted (v1, v2, v3...), ensuring a transparent audit trail of a concept's growth.
- **Execution Status Layer**: Ideas progress through a defined lifecycle: `Thinking` → `Refining` → `Building` → `Launched`.

### 🗺️ Tactical Map Intelligence 2.0
A geolocation-aware discovery engine that visualizes the innovation landscape.
- **Hybrid View**: Distinguish between standalone Ideas (Indigo) and defining Problems (Purple) across the globe.
- **Contextual Density**: Tactical heatmaps identify "Innovation Hot Zones" where high-severity problems meet active solutions.
- **Radius-Based Discovery**: Scan local vectors from 5km to global tiers to find where you can have the most impact.

### 🤝 Collaborative Team Layer
Build teams through a structured, architect-led approval flow.
- **Architect Controls**: Project leads have full authority over who joins their team.
- **Automated Sync**: Upon operative approval, the system initializes private discussion threads automatically to bridge the gap from request to collaboration.

---

## 🛡️ Industrial Grade Tech Stack

Built for performance and scalability:
- **Core**: Next.js 16 (App Router / Turbopack)
- **State & Data**: Firebase Firestore (Real-time sync)
- **Visuals**: Framer Motion & Tailwind CSS
- **Intelligence**: Leaflet.js with custom Tactical Heat Layers
- **Real-time Comms**: Integrated messaging layer with automated triggers

## 🛠️ Getting Started

### Prerequisites
- Node.js 18+
- Firebase Project setup

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/arun6832/Inzly.git
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set up environment variables (`.env.local`):
   ```env
   NEXT_PUBLIC_FIREBASE_API_KEY=your_key
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_domain
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_id
   ```
4. Run the development server:
   ```bash
   npm run dev
   ```
